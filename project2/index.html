Thinking Like a Programmer ...

There are five key ways to think like a programmer:

  • Procedural Thinking which involves creating simple instructions for a computer to follow.
  • Abstract Thinking which involves finding commonalities among different things.
  • Systems Thinking which may sound intimidating, but this involves breaking down a large problem into smaller tasks.
  • Technological Empathy involves being able to understand a computer's actions.
  • Debugging which allows programmers to fix problems.

CSS allows a user to style their webpage. Jacques gave the perspective of viewing a webpage in a series of boxes which helped me internalize a layout for simple to complex design.

CSS Lingo

Style: this is lingo that is pretty vague, so here's what it can mean:

  • A HTML element
  • A HTMLÂ attribute
  • The look of a website
  • Or a verb as referenced by the Udacity team

Rule: this describes the value an attribute should have

Property: whatever it is you want to modify, update or change

Value: this is the best friend to an attribute

Selector: this is used for elements that partner with classes or attributes

Class: a group of things that can be styled in bulk

ID: this is just an unique name for lone sheep, aka lone elements that don't belong

CSS Styling Methods

Method 1

For those folks out there with minimum time, you can style quickly in the head of your HTML file. Remember this code will live in the head element of your HTML document!

Method 2

For those folks who are a bit more organized, you're going to want to link your CSS stylesheet to your HTML stylesheet in the head. If you follow the steps below, you should be good to go in terms of styling.

  • 1. Write all HTML
  • 2. Write all of your CSS in a different file labeled ".css"
  • 3. Within the head of the HTML file, add a link to the stylesheet
  • 4. Don't forget to add attributes :)

Method 3

For messy and disorganized people, you should probably use this method. You should probably turn back now because you are essentially going to create more work for yourself by writing DUPLICATE code. Ain't nobody got time... In this method, you would update every attribute for EVERY HTML element! Personally, I would go with Method 2!
